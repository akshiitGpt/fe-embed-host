Embeddable UI Components

Drop-in components that can be embedded into any app via a single script tag. Each component supports two modes: direct DOM mount (like Clerk) or iframe isolation.

6 Components 2 Embed Modes Real API Calls Zero Dependencies
Method 1

Direct Embed (SDK)

The SDK mounts components directly into your DOM. Full access to host context, callbacks, and styling.

  • Renders in host DOM
  • JS callbacks (onLoad, onError)
  • Shared auth & context
  • Smallest footprint
Method 2

iframe Embed

Each component has a standalone HTML page that can be loaded in an iframe. Full isolation.

  • Complete CSS/JS isolation
  • postMessage communication
  • Independent deployment
  • Best for third-party use

Quick Start

Add one script tag, then mount any component

<!-- Load the SDK --> <script src="https://akshiitgpt.github.io/fe-embed-widget/sdk.js"></script> <!-- Mount components anywhere --> <div id="my-widget"></div> <script> EmbedWidget.Analytics('#my-widget'); </script>
// Mount any component with options const profile = EmbedWidget.UserProfile('#container', { userId: 1, onLoad: (data) => console.log('User loaded', data), }); // Control it programmatically profile.refresh(); // re-fetch data profile.unmount(); // remove from DOM // Available: Analytics, UserProfile, ActivityFeed, // PricingTable, LoginForm, DataTable
// iframe any standalone widget page EmbedWidget.iframe('#container', { page: 'widgets/user-profile.html', onLoad: (data) => console.log('Loaded'), }); // Or use raw iframe: <iframe src="https://akshiitgpt.github.io/fe-embed-widget/widgets/login-form.html" style="width:100%;height:500px;border:none" ></iframe>
