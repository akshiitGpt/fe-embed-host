Embed SDK — Two Ways

This dashboard demonstrates both embedding methods, like Clerk.com components. The Analytics tab is available in iframe mode (isolated, hosted page) and direct embed mode (SDK mounts into your DOM). Both call real APIs and render the same dashboard — zero widget source code in this repo.

Method 1

iframe Embed

Loads the full hosted page inside an iframe. Complete isolation between host and embedded frontend.

  • Full CSS/JS isolation
  • Separate deployment lifecycle
  • Communication via postMessage
  • Best for: untrusted or third-party content
Method 2

Direct Embed (SDK)

A JavaScript SDK mounts the component directly into your DOM. Like Clerk's <SignIn /> components.

  • Renders in host DOM, no iframe
  • Direct JS callbacks (onLoad, onError)
  • Inherits host context & auth
  • Best for: first-party embedded dashboards

// Method 1: iframe embed const widget = EmbedWidget.iframe('#container', { title: 'Analytics Dashboard', onLoad: (data) => console.log('Loaded', data), onError: (err) => console.error(err), }); widget.refresh(); // trigger reload widget.unmount(); // remove widget
// Method 2: direct embed — mounts into your DOM const widget = EmbedWidget.mount('#container', { title: 'Analytics Overview', onLoad: (data) => console.log('Loaded', data), onError: (err) => console.error(err), onFilterChange: (range) => console.log('Filter:', range), }); widget.refresh(); // re-fetch and re-render widget.unmount(); // clean up
<!-- Add this to your HTML --> <script src="https://akshiitgpt.github.io/fe-embed-widget/sdk.js"></script> <!-- Then use either method --> <div id="analytics"></div> <script> EmbedWidget.mount('#analytics'); // direct // OR EmbedWidget.iframe('#analytics'); // iframe </script>
